## 2. 项目结构

### 2.1 目录结构
```
src/
├── api/             # API客户端和端点定义
├── assets/          # 静态资源
├── components/      # 组件（按功能分类）
│   ├── auth/        # 认证相关组件
│   ├── common/      # 通用组件
│   ├── editor/      # 编辑器组件
│   └── front/       # 前台页面组件
├── constants/       # 常量定义
├── hooks/           # 自定义Hooks
├── layouts/         # 布局组件
├── pages/           # 页面组件
│   ├── admin/       # 后台管理页面
│   └── front/       # 前台页面
├── routes/          # 路由配置
├── services/        # 业务逻辑服务
├── store/           # 状态管理
├── types/           # TypeScript类型定义
├── utils/           # 工具函数
├── App.tsx          # 应用入口
└── main.tsx         # 主文件
```

### 2.2 命名规范
- **文件命名**: 使用 PascalCase 或 kebab-case
- **组件命名**: 使用 PascalCase
- **函数命名**: 使用 camelCase
- **变量命名**: 使用 camelCase
- **常量命名**: 使用 UPPER_CASE
- **类型命名**: 使用 PascalCase

### 2.3 枚举、公共类、常量使用规范
- **枚举类**: 前端应创建与后端对应的枚举类，确保枚举值和命名与后端保持一致
- **公共类**: 对于前后端共用的工具类或辅助类，前端应创建对应的 TypeScript 实现，与后端保持一致
- **常量**: 前端应定义与后端一致的常量，包括 API 路径、状态码、配置项等
- **避免重复**: 在创建枚举、公共类或常量之前，应先查看项目中是否已存在或已定义，避免重复和冗余
- 这样可以确保前后端行为一致，减少沟通成本，提高代码的可维护性

## 3. 状态管理

### 3.1 Zustand 使用规范
- 使用 create 函数创建 store
- 使用 persist 中间件进行状态持久化
- 为每个 store 创建相应的选择器
- 导出 store 实例，用于在非 React 组件中访问
- 使用不可变更新模式
- 避免在 store 中直接操作 DOM

### 3.2 状态管理最佳实践
- 避免全局状态过多
- 状态应该尽可能局部化
- 使用 useCallback 缓存函数，减少不必要的重渲染
- 使用 useMemo 缓存计算结果，减少不必要的计算
- 合理使用 useEffect，避免无限循环

## 4. 组件开发

### 4.1 组件设计原则
- 单一职责原则: 每个组件只负责一个功能
- 可复用性: 设计可复用的组件
- 可测试性: 组件应该易于测试
- 性能优化: 避免不必要的重渲染

### 4.2 组件结构
- 使用函数式组件
- 使用 TypeScript 类型定义
- 使用 React Hooks
- 合理使用 props 和 state
- 合理使用 context

## 5. API 调用

### 5.1 服务层设计
- 将 API 调用封装在 services 目录下
- 使用 axios 进行 HTTP 请求
- 统一处理错误和响应
- 使用 async/await 语法

### 5.2 API 调用最佳实践
- 建议使用 try/catch 捕获错误，确保错误能够被妥善处理
- 合理处理加载状态
- 合理处理错误状态
- 避免在 useEffect 中直接调用异步函数

### 5.3 前后端数据结构一致性
- 前端应与后端保持数据结构一致，包括请求参数和返回数据
- 对于复杂的请求参数，前端应创建对应的 TypeScript 类型，与后端的 DTO 或实体类保持一致
- 对于后端返回的数据，前端应创建对应的 TypeScript 类型，与后端的 VO 类型保持一致
- 这样可以提高代码的可维护性，减少类型错误，便于团队协作

## 7. 性能优化

### 7.1 代码分割
- 使用 React.lazy 进行路由级别的代码分割
- 使用动态 import 进行组件级别的代码分割

### 7.2 渲染优化

- 使用 React.memo 缓存组件
- 避免在渲染过程中创建新对象

### 7.3 网络优化
