# InkStage 前端项目介绍

## 项目概述

InkStage 是一个现代化的内容创作与分享平台，提供文章发布、阅读、评论、收藏等功能。前端项目基于 React 19.2.0 + TypeScript 构建，采用现代化的技术栈和开发流程，为用户提供流畅、美观的交互体验。

## 技术栈

| Axios | 1.13.5 | HTTP 客户端 |

## 代码风格

### 1. 命名规范

- **组件名**：使用 PascalCase（如 `ArticleCard`）
- **函数名**：使用 camelCase（如 `fetchArticleDetail`）
- **变量名**：使用 camelCase（如 `articleId`）
- **常量**：使用 UPPER_SNAKE_CASE（如 `API_ENDPOINTS`）
- **类型名**：使用 PascalCase（如 `ArticleType`）

### 2. 代码组织

- **组件化**：将 UI 拆分为可复用的组件
- **关注点分离**：业务逻辑与 UI 分离
- **代码分割**：使用 React.lazy 实现按需加载
- **类型定义**：为所有数据结构添加 TypeScript 类型

### 3. 样式规范

- 使用 Tailwind CSS 进行样式开发
- 遵循响应式设计原则
- 保持一致的设计语言

### 4. 性能优化

- 使用 React.memo 优化组件渲染
- 使用 useCallback 和 useMemo 减少不必要的计算
- 图片懒加载
- 代码分割

## 部署方案

### 1. 静态资源部署

- 使用 nginx 作为静态资源服务器
- 配置 gzip 压缩和缓存策略

### 2. 容器化部署

- 使用 Docker 容器化应用
- 使用 docker-compose 管理多服务

### 3. 反向代理配置

- 配置 nginx 反向代理到后端 API
- 解决跨域问题

## 与后端集成

### 1. API 调用方式

- **API 客户端**：使用 Axios 封装的 `apiClient.ts` 进行 API 调用
- **API 端点**：集中管理在 `apiEndpoints.ts` 文件中
- **请求拦截**：统一处理认证令牌、错误处理等
- **响应处理**：统一处理 API 响应格式，提取数据和错误信息

### 2. 状态管理

- **全局状态**：使用 Zustand 管理全局状态，如用户信息、文章数据等
- **局部状态**：使用 React useState 和 useReducer 管理组件局部状态
- **数据缓存**：使用 React Query 缓存 API 响应，提高性能

### 3. 认证流程

- **登录认证**：通过 `/api/front/auth/login` 接口获取 JWT 令牌
- **令牌存储**：使用 localStorage 存储认证令牌
- **令牌刷新**：实现令牌自动刷新机制
- **路由保护**：使用 `RouteGuard` 组件保护需要认证的路由

### 4. 实时通信

- **WebSocket**：使用 STOMP.js 和 SockJS 实现 WebSocket 连接
- **实时通知**：接收后端推送的实时通知
- **消息处理**：处理不同类型的实时消息

## 性能优化策略

### 1. 前端优化

- **代码分割**：使用 React.lazy 和 Suspense 实现代码分割
- **组件优化**：使用 React.memo、useCallback、useMemo 优化组件渲染
- **图片优化**：使用懒加载、适当尺寸的图片
- **网络优化**：使用 HTTP/2、缓存策略

### 2. 数据优化

- **数据缓存**：使用 React Query 缓存 API 响应
- **分页加载**：实现分页加载，减少一次性加载的数据量
- **防抖节流**：对搜索、滚动等事件进行防抖节流处理

### 3. 构建优化

- **Tree Shaking**：移除未使用的代码
- **代码压缩**：压缩 JavaScript、CSS 文件
- **资源优化**：优化字体、图片等静态资源
